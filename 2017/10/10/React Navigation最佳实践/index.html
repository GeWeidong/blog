React Navigation最佳实践

Posted on 2017-10-10 | In javascript | | Visitors: | 阅读数

本文是基于最新的react-navigation^1.0.0-beta.13来总结的。

1.为啥使用react-navigation？

  • react-navigation比Navigator更加流畅，体验要好一些；

  • 编码更规范，有层次感，易于管理维护；

  • 官方推荐，没有发现明显的bug，且用户量大，社区活跃，文档齐全。

  • 官方文档

  • 一个用react-navigation写的较大型的app，结合redux，参考性强

  • 前端大神写的，必看!

3.功能

  • StackNavigator
    类似顶部导航条，用来跳转页面和传递参数。

  • TabNavigator
    类似底部标签栏，用来区分模块。

  • DrawerNavigator
    抽屉，类似从App左侧滑出一个页面，==实际实用性不大==，我们一般用到的是DrawerLayoutAndroid或是其他开源组件，所以在这里不做讲解。
